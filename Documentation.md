# Структура курса на Stepik

## Иерархия
```
Course (Курс)
├── Section 1 (Модуль)
│   ├── Unit → Lesson 1.1 (Урок)
│   │              └── Step 1, Step 2...
│   ├── Unit → Lesson 1.2
│   └── Unit → Lesson 1.3
├── Section 2
│   └── Unit → Lesson 2.1
└── Section 3
    └── Unit → Lesson 3.1
```

## Ключевые объекты и их связь

### Course (Курс)
**GET** `/api/courses/{course_id}`
```json
{
  "id": 253149,
  "title": "Название",
  "sections": [111, 222]  // IDs секций
}
```

### Section (Модуль)
**GET** `/api/sections/{section_id}`
```json
{
  "id": 111,
  "title": "Модуль 1",
  "position": 1,
  "units": [333, 444]  // IDs юнитов
}
```

### Unit (Связка) — САМОЕ ВАЖНОЕ
**GET** `/api/units/{unit_id}`
```json
{
  "id": 333,
  "section": 111,  // К какой секции относится
  "lesson": 555,   // Какой урок содержит
  "position": 1    // Позиция урока в секции (1.1, 1.2...)
}
```

### Lesson (Урок)
**GET** `/api/lessons/{lesson_id}`
```json
{
  "id": 555,
  "title": "Урок 1.1",
  "steps": [777, 888]  // IDs шагов
}
```

## Как получить структуру курса за 4 шага

### Шаг 1. Курс → Секции
```
GET /api/courses/253149
```
Получаем `sections`: [111, 222]

### Шаг 2. Секции → Юниты
```
GET /api/sections?ids[]=111&ids[]=222
```
Из каждой секции берём `units`

### Шаг 3. Юниты → Связка секций и уроков
```
GET /api/units?ids[]=333&ids[]=444...
```
Из юнитов узнаём:
- Какой урок (`lesson`) к какой секции (`section`)
- Позицию урока (`position`) внутри секции

### Шаг 4. Юниты → Уроки
```
GET /api/lessons?ids[]=555&ids[]=666...
```
Получаем названия уроков и их шаги

## Важно!
- **Lesson не знает, в какой он секции** — эту связь даёт Unit
- **Позиция урока в меню** (1.1, 1.2) берётся из `Unit.position`
- API всегда возвращает объекты в списке, даже один:
```json
{"courses": [{...}]}  // даже для одного курса
```
